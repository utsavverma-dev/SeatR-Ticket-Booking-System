# SeatR - Premium Ticket Booking System

SeatR is a modern, full-stack event and ticket booking platform designed with a sleek aesthetic. 

This repository contains the complete source code for the project, cleanly separated into a Spring Boot Backend and a React Frontend.

## 📂 Project Structure

This project has been structured to adhere strictly to submission guidelines, ensuring no unnecessary or temporary files (like `node_modules`, `.env`, build artifacts, or editor configurations) are included in the final committed codebase.

The repository contains two main directories:

- `/backend` - The Spring Boot (Java) REST API.
- `/frontend` - The React (Vite) Single Page Application.

*Note: Only the essential application files and configuration necessary to run the project are included. Dependencies are kept native and minimal where possible.*

---

## 🏗️ Architecture & Tech Stack

### 1. Backend (Spring Boot)
The backend serves as a secure RESTful API that handles business logic, database operations, user authentication, and email services.

- **Framework**: Java 17+, Spring Boot 3.x
- **Database**: MySQL (via Spring Data JPA / Hibernate)
- **Security**: Spring Security with JWT (JSON Web Tokens)
- **Core Features**:
  - Role-based Access Control (Admin, Organiser, Customer).
  - Secure endpoints for Venues, Events, Tickets, and User Management.
  - Help Centre integration (automated email delivery).

### 2. Frontend (React + Vite)
The frontend is a fast, responsive Single Page Application offering an immersive UI.

- **Framework**: React 18, Vite
- **Styling**: TailwindCSS
- **Routing**: React Router DOM
- **Core Features**:
  - Dynamic Dashboards tailored to the user's role.
  - Interactive popup modals (e.g., Help Centre).
  - Toast notifications and state management via Hooks.

## 🚀 Getting Started

To run the application locally, you must start both the backend and frontend servers. **Ensure the application runs without errors by following these steps.**

### Prerequisites
- **Java 17 or 21** installed (`JAVA_HOME` configured).
- **Maven** installed.
- **Node.js** (v18+) and **npm** installed.
- **MySQL Server** installed and running.

---

### Part 1: Running the Backend

1. **Configure Database**:
   - Ensure your MySQL server is running locally.
   - Create a database named `ticket_booking` (or update `application.properties` to match your local setup).
   - Verify the database credentials in `backend/src/main/resources/application.properties` are correct (default is usually `root` / `password`).

2. **Start the Application**:
   Open a terminal and navigate to the backend directory:
   ```bash
   cd backend
   mvn spring-boot:run
   ```
   The backend will start and run on **http://localhost:8080**.

---

### Part 2: Running the Frontend

1. **Install Dependencies**:
   Open a *new* terminal window and navigate to the frontend directory:
   ```bash
   cd frontend
   npm install
   ```
   *(Note: The `node_modules` folder generated by this step is excluded from the repository as per submission guidelines).*

2. **Start the Development Server**:
   ```bash
   npm run dev
   ```
   The frontend will compile instantly and run on **http://localhost:5173**. Open this URL in your browser to interact with the application.
